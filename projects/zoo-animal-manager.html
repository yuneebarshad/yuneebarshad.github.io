Overview

ROLE: Senior Game Developer — project architecture & core systems

A mobile zoo management simulation built from scratch in Unity for Android, where players expand a zoo by unlocking exhibits, purchasing and caring for animals, feeding them through a physics-based interaction system, completing quests, and managing a full in-game economy. I architected the core gameplay systems — exhibit progression, animal AI, visitor navigation, quests, and the currency economy — and integrated third-party SDKs for monetization, analytics, and mobile tooling to bring it to a production-ready, live release.

Gameplay Loop

Earn currency→ Unlock exhibit→ Buy animal→ Feed & care→ Complete quest→ Expand zoo

Key Highlights

  • Built the complete gameplay loop from scratch in Unity, from exhibit unlocking through quest-driven progression.
  • Implemented a persistent, encrypted local save system for currency, using XOR-style encoding to protect player economy data.
  • Developed AI-driven animals with hunger, survival, and food-detection behavior tied to a physics-based feeding mechanic.
  • Created NavMesh-based visitor NPC movement with obstacle avoidance and collision recovery.
  • Integrated Unity IAP, AppLovin MAX, Firebase Analytics, and Remote Config into a production monetization pipeline, including exponential backoff retry logic on ad requests.
  • Optimized Android performance with device-memory-based quality scaling, tuning camera far clip and LOD bias across low- and mid-range devices.

Core Systems

✓ Exhibit & Animal Management (Script)

Handles unlocking exhibits, purchasing animals, and spawning them into the correct enclosure.

  • Loads and saves exhibit unlock/purchase state from persistent JSON on session start
  • Swaps locked-area prefabs for active arena prefabs at runtime
  • Wires each unlocked exhibit to animal spawn points, food points, and cutscene triggers
✓ Feeding & Animal AI (Script)

A physics-based feeding mechanic paired with hunger-driven animal behavior, rather than a tap-to-feed button.

  • Cannon-based aiming with projectile arc preview and turret rotation, using Rigidbody physics for the food throw
  • Active food tracking per exhibit, with AI notification when food lands so animals detect and respond to it
  • Hunger and survival values tick down per animal, with feed state and remaining food count persisting across sessions
✓ NPC Visitor System (Script)

Populates the zoo with visitors that make it feel alive rather than static.

  • NavMeshAgent-driven pathing between destinations
  • Animation state triggers for walk/idle transitions
  • Obstacle avoidance tuning and stuck/collision recovery
✓ Quest & Level Progression (Script)

Ties quests and levels together into the loop that keeps players moving forward.

  • Quest tracking across feeding, cleaning, and exhibit-unlock tasks
  • Level-gated content unlocks for new exhibits, animals, and features
  • Tutorial flow and minimap support for onboarding and navigation
✓ Economy & Currency (Script)

A dedicated economy layer underneath the whole game, not just a coin counter.

  • Starting cash, currency add/deduct logic, and separate ticket currency for zoo entry
  • Encrypted local currency saves using XOR-style encoding, protecting player progress from casual tampering
  • Animated coin collection and UI sync using DOTween/LeanTween across multiple panels
  • Rewarded-ad and IAP currency flows feeding directly into the same economy system
✓ Monetization & Live Ops (Third-Party Integration)

The production layer that turns a finished build into a live, revenue-generating app.

  • AppLovin MAX integration — rewarded, interstitial, banner, and app-open ad formats, with exponential backoff retry logic on failed requests
  • Unity IAP for consumable currency packs and non-consumable purchases, including restore-purchase handling
  • Firebase Analytics and Remote Config for event tracking, ad revenue impression logging, and remotely-tunable gameplay values
  • Google Play plugin integration for store-side services
✓ Save System & Android Optimization (Script)

The framework layer underneath the whole game: what makes it a mobile game framework, not a prototype.

  • Persistent JSON-based save system spanning animals, coins, food, quests, levels, and exhibit unlocks
  • Device-memory-aware quality scaling — shorter camera far clip on 3GB devices, extended range on 4GB+, with LOD bias tuned per device tier
